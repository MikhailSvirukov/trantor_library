## как сделать atomic broadcast?
1. несбойные узлы доставят сообщение и каждое сообщение будет доставлено
2. общий порядрк доставки

* доставить (reliable broadcast) + порядок (consensus)
* consensus: каждый узел реализует синхронный алгоритм с одинаковым входом и что-то возвращаем; узлы из общего что-то выбирают: что-то из предложенного (validity) + все узлы выбирают одно и то же (agreement) + алгоритм завершается (termination); как выбрать общее значение?
* reliable broadcast: отправляем сообщение всем - каналы не теряют сообщения - когда узел получает сообщение первый раз, он делает re-transmission (в реальности - вероятностные алгоритмы)
* серия консенсусов: 
* операция AB: отправка всем сообщения RB -> на каждом узле обработчик помещает сообщение в множество + множество сообщений, для которых вызван обработчик доставки; фоновый процесс с раундами (пока есть сообщения полученные через RB, но не отправленные, используем консенкус (предлагаем сообщения и получаем какое-то общее значение)) -> на всех вызываем обработчик доставки для этого сообщения и убираем сообщение в A 
* *задача консенсуса решена (не важно как)*
* каждый консенсус выбирает новую задачу
* отказоустойчивость SRM сводится к отказоустойчивости consensus
* почему выбираем не 1 сообщение, а пачка сообщений? иначе сломается validity (чтобы какое-то сообщение не проигрывало всегда)
