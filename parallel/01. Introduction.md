### Параллельное программирование

* *Параллелизм* - подход к вычислениям, при котором некоторые участки программ могут быть выполнены независимо и одновременно
* *Конкурентность* - одновременный доступ к некоторому ресурсу
* *Асинхронность* - подход к вычислениям, при котором вычисление не дожидается завершения задачи и обрабатывает результат после выполения задачи
Идея общая: ускорить вычисления, сокращая время без непосредственного вычисления

## Параллелизм

Уровни параллелизма:
1. Битовый (процессор работает на уровне регистра - над всеми битами сразу, и прочие крутые оптимизации схем)
2. Инструкций (pipeline, superscalar)
3. Данных (всевозможные инструкции с возможностью исполнения сразу с набором данных (например, векторные инструкции для сложения массивов), aka SIMD-инструкции)
4. Задач (в первом приближении - просто выполнеие каких-то более продолжительных кусков кода)

## Таксономия Флинна

Классификация вычислителей с точки зрения множественности инструкций и данных, который способен выполнять в момент времени
* SISD
* SIMD (x86 - MMX, SSE, AVX, матричные процессоры)
* MISD
* MIMD (многоядерный процессор, VLIW)
* SIMT (T - thread)

## Закон Амдала
оценка эффективности распараллеливания

Пусть есть вычислительная задача с фиксированным объемом (набор инструкций для выполнения), N - процессоров вычислителя, s - доля последовательных вычислений, p - доля параллельных вычислений (s+p=1)
Ti - время на i процессорах
S = T1 / TN = T1 * (s+p) / (s * T1 + p * T1 / N) = 1 / (s + p / N) = 1 / (s + (1 - s) / N) 
**SpeedUp (S) стремится к 1 / s !!!**
но: 
* вообще-то параллелить конечный набор инструкций бесконечно невозможно
* вообще-то есть расходы на поддержание параллельности (GLL)

## Закон Густавсона ( -Барсиса)
Пусть время фиксированно (Vi),  N - процессоров вычислителя, , s - доля последовательных вычислений, p - доля параллельных вычислений (s+p=1)

можно на каждом потоке выполнить параллельную задачу целиком:
S = VN / V1  = (V1 * s + V1 * p * N) / V1 = s + p * N = N + (1 - N) * s

### Средства организации параллелизма
1. *Нить* (fiber, coroutines) (кооперативная многозадачность, в которой явно в коде происходит переключение между нитями на одном потоке)
2. *Многопоточность*
3. *Многопроцессность* (сильная независимость из-за разных адресных программ; по сравнению с этим, задачи в многопоточности не обладают сильной независимостью - частенько они ломают друг другу данные или конкурируют)
4. *Кластерность*
5. *Децентрализованные вычисления*

 \+ всякие акселлераторы (какое-то время был FPU, GPGPU, FPGA, ASIC)
